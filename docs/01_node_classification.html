3 Part A — Origin Processing (Network Nodes)

A1 — Functional Type Flags

nodes_on(gdf_nodes, gdf_links, ft_codes) returns True for any node whose N appears as endpoint A or B of at least one link whose FT code is in ft_codes.

Flags are non-exclusive — a node touching both a freeway and a ramp link will be True for both Freeway and Ramp. Mutual exclusion is enforced later in the topology cascade (§A3).

A2 — Link Count, Neighbor Count, and Direction

count_links adds LinkCount — raw endpoint references per node, used for the distribution chart and diagnostics.

count_neighbors adds NeighborCount — the number of distinct neighboring nodes. This is the topologically correct pseudonode test: LinkCount breaks on bidirectional roads stored as two directional records (a pseudonode appears 4 times → LinkCount=4, not 2), while NeighborCount is always 2 for a true shape vertex regardless of storage convention.

A node is an is_pseudo shape vertex if and only if both conditions hold: 1. NeighborCount == 2 — lies on a single path between two real junctions. 2. LinkClassCount == 1 — all connected links share the same functional class. A node where a freeway meets a surface street has NeighborCount=2 but LinkClassCount=2, so it is a real class-transition junction and must be snapped.

assign_node_directions produces two direction columns (see docstring): - link_directions — freeway-priority cascade; used in passes (a) and (c). - fw_directions — freeway links only, majority-vote; used in pass (b) gore nodes.

    gdf_links[["A"]].assign(cls=_link_class.values).rename(columns={"A": "N"}),
    gdf_links[["B"]].assign(cls=_link_class.values).rename(columns={"B": "N"}),
], ignore_index=True)
_class_counts = _stacked_class.groupby("N")["cls"].nunique()
gdf_nodes["LinkClassCount"] = gdf_nodes["N"].map(_class_counts).fillna(0).astype(int)

# Count distinct FT codes per node.
# Stricter than LinkClassCount: nodes where FT changes (e.g. FT=32→33) register
# LinkFTCount=2 even though both belong to the freeway class.  Used in the future

# speed or facility sub-type changes — dissolving those nodes would corrupt TDM impedance.
_stacked_ft = pd.concat([
    gdf_links[["A", FT_COL]].rename(columns={"A": "N"}),
    gdf_links[["B", FT_COL]].rename(columns={"B": "N"}),
], ignore_index=True)
_ft_counts = _stacked_ft.groupby("N")[FT_COL].nunique()
gdf_nodes["LinkFTCount"] = gdf_nodes["N"].map(_ft_counts).fillna(0).astype(int)

# Pseudonode: shape vertex on a single homogeneous path — no physical snap target.
# Uses LinkFTCount (not LinkClassCount): UGRC/UDOT centerlines are segmented at

A3 — Node Topology Classification

Eight mutually-exclusive node_type labels. Transit is evaluated first and takes priority over all road-class combinations. The remaining seven labels derive from the full combinatorics of the (Freeway, Ramp, Surface) flags.

HOV/managed-lane nodes (FT 37-39) were filtered at load time, so they contribute no flags; those nodes fall to “surface” and are not snapped. Pseudonodes (LinkCount == 2) are shape vertices with no physical counterpart.

Show the code 
no_pseudo = ~gdf_nodes["is_pseudo"]

is_fwy_node     = gdf_nodes["Freeway"]      & no_pseudo
is_ramp_node    = gdf_nodes["Ramp"]         & no_pseudo
is_surface_node = gdf_nodes["Surface"]      & no_pseudo
is_transit_node = gdf_nodes["FixedTransit"] & no_pseudo

# Full combinatorics of the three link classes → 7 node types.
# Transit is evaluated first — station nodes get "transit" regardless of any
# road-class links they may also touch.
# Most-specific road combinations (all three flags) are evaluated next.
gdf_nodes["node_type"] = np.select(
    [
        is_transit_node,                                  # fixed rail station
        is_fwy_node &  is_ramp_node &  is_surface_node,  # fwy+ramp+sf → at-grade interchange
        is_fwy_node &  is_ramp_node & ~is_surface_node,  # fwy+ramp    → elevated gore

B2 — Classify Segments

Three mutually exclusive boolean flags per segment. Primary authority is DOT_FCLASS for freeway identification; DOT_RTNAME (UDOT LRS) supplements with explicit route numbers and ramp/CD facility characters. ONEWAY in (1, 2) elevates divided Principal Arterials into the freeway class (domain CVDomain_OneWay: 0=Two-way, 1=One-way direction of arc, 2=One-way opposite direction of arc).

DOT_RTNAME layout (11 characters, zero-padded):
  [0:4]  Route number      e.g. "0015" = I-15, "0154" = Bangerter
  [4]    LRS direction     "P" = primary (NB/EB),  "N" = negative (SB/WB)
  [5]    Facility type     " " = mainline, "R" = ramp, "C" = collector-distributor
  [6:]   Sequence/suffix   varies

These flags flow into every endpoint pool and type-tier decision, making this the primary place to audit when snapping results look wrong.

## Part A — Origin Processing (Network Nodes)

### A1 — Functional Type Flags

`nodes_on(gdf_nodes, gdf_links, ft_codes)` returns True for any node whose
N appears as endpoint A or B of at least one link whose FT code is in `ft_codes`.

Flags are **non-exclusive** — a node touching both a freeway and a ramp link
will be True for both `Freeway` and `Ramp`. Mutual exclusion is enforced later
in the topology cascade (§A3).

```{python}
# ── Three mutually exclusive TDM link classes (2027 network year) ─────────
#
# FT_2027 code reference
# ─────────────────────────────────────────────────────────────────────────
# Class 1 — Freeway Mainline (FW_CODES):
#   12-15  Expressway (limited-access, divided)
#   22-26  MM freeway (posted 55-75 mph, MM series)
#   32-36  Freeway    (posted 55-75 mph, Fwy series)
#   40     Express / managed-access freeway
#   FT=2 AND DIRECTION=1: directional Principal Arterial links representing
#         physically separated roadways (e.g. divided expressways coded as PA).
#
# Class 2 — Ramps and CDs (RAMP_CODES):
#   20/30  Loop ramps  |  28/29  MM on/off ramps  |  41/42  Fwy on/off ramps
#   21/31  CD roads (collector-distributor geometry)
#
# Class 3 — Surface Streets (SURFACE_CODES):
#   2/3  Principal/Minor Arterial  |  4/5  Major/Minor Collector  |  6/7  Local/Unpaved
#   FT=2 links with DIRECTION=1 are excluded (they belong to Class 1 above).
#
# Filtered out at load time (not present in gdf_links):
#   1  Centroid connector (zone-to-network tap, not a physical road)
#   37 HOV lane  |  38 Managed lane  |  39 Managed-lane access point
#   Removed before any processing so they cannot inflate LinkCount/NeighborCount,
#   corrupt the pseudonode LinkClassCount check, pollute the direction cascade,
#   or produce spurious surface-type node classifications.
#
# Loaded but excluded from the three road classes:
#   70/80 Fixed transit  |  N ranges 10000-19999 and 50000-59999 (station nodes)
#   Transit geometry is handled separately in the FixedTransit snapping pass.
# ─────────────────────────────────────────────────────────────────────────

FT_COL = "FT_2027"

# DIRECTION=1 flag: handles both numeric 1 and string "1" encodings
_oneway = gdf_links["ONEWAY"].astype(str).str.strip() == "1"

# Mutually exclusive link masks (LN_2027 > 0 already filtered at load time)

# 1. Freeway includes interstates and expressways/principal arterials that have directional travel
FWY_LINK_MASK = gdf_links[FT_COL].isin({22, 23, 24, 25, 26, 32, 33, 34, 35, 36, 40}) | (
    gdf_links[FT_COL].isin({2, 12, 13, 14, 15}) & _oneway
)

# 2. Ramp and Collector-Distributor Roads share the same mask
RAMP_LINK_MASK = gdf_links[FT_COL].isin({20, 21, 28, 29, 30, 31, 41, 42})

# 3. Centroid Connectors (FT = 1)
CC_LINK_MASK = gdf_links[FT_COL] == 1

# 4. HOV and Managed Lanes (FT = 37, 38, 39)
MANAGED_LINK_MASK = gdf_links[FT_COL].isin({37, 38, 39})

# 5. Remaining Links get assigned as Surface Links
SURFACE_LINK_MASK = ~(FWY_LINK_MASK | RAMP_LINK_MASK | CC_LINK_MASK | MANAGED_LINK_MASK)

# FW_CODES: FT codes used for the freeway-priority direction cascade.
# FT=2+DIRECTION=1 is classified as freeway but excluded here because its
# DIRECTION value is a one-way flag, not a cardinal (NB/SB/EB/WB).
# Direction matching for those links is deferred to the geometric heading
# approach planned for the ramp refactor (Request 3).
FW_CODES = {12, 13, 14, 15, 22, 23, 24, 25, 26, 32, 33, 34, 35, 36, 40}

gdf_nodes["Freeway"] = nodes_on(gdf_nodes, gdf_links[FWY_LINK_MASK])
gdf_nodes["Ramp"] = nodes_on(gdf_nodes, gdf_links[RAMP_LINK_MASK])
gdf_nodes["Surface"] = nodes_on(gdf_nodes, gdf_links[SURFACE_LINK_MASK])
gdf_nodes["FixedTransit"] = (
    nodes_on(gdf_nodes, gdf_links, {70, 80})
    | gdf_nodes["N"].between(10_000, 19_999)  # TRAX station nodes
    | gdf_nodes["N"].between(50_000, 59_999)  # FrontRunner station nodes
)

flag_cols = ["Freeway", "Ramp", "Surface", "FixedTransit"]
print("Flag counts:")
print(gdf_nodes[flag_cols].sum().rename("NodeCount").to_frame().to_string())
print(
    f"\nLinks by class — Freeway: {FWY_LINK_MASK.sum():,}  "
    f"| Ramp: {RAMP_LINK_MASK.sum():,}  "
    f"| Surface: {SURFACE_LINK_MASK.sum():,}"
)
```

### A2 — Link Count, Neighbor Count, and Direction

`count_links` adds `LinkCount` — raw endpoint references per node, used for the
distribution chart and diagnostics.

`count_neighbors` adds `NeighborCount` — the number of **distinct** neighboring
nodes. This is the topologically correct pseudonode test: `LinkCount` breaks on
bidirectional roads stored as two directional records (a pseudonode appears 4
times → LinkCount=4, not 2), while NeighborCount is always 2 for a true shape
vertex regardless of storage convention.

A node is an `is_pseudo` shape vertex if and only if both conditions hold:
1. **NeighborCount == 2** — lies on a single path between two real junctions.
2. **LinkClassCount == 1** — all connected links share the same functional class.
   A node where a freeway meets a surface street has NeighborCount=2 but
   LinkClassCount=2, so it is a real class-transition junction and must be snapped.

`assign_node_directions` produces two direction columns (see docstring):
- `link_directions` — freeway-priority cascade; used in passes (a) and (c).
- `fw_directions`   — freeway links only, majority-vote; used in pass (b) gore nodes.

```{python}
gdf_nodes = count_links(gdf_nodes, gdf_links)
gdf_nodes = count_neighbors(gdf_nodes, gdf_links)
gdf_nodes = assign_node_directions(gdf_nodes, gdf_links, freeway_ft_codes=FW_CODES)

# Assign one class label per link using the already-defined mutually exclusive masks.
_link_class = pd.Series("other", index=gdf_links.index)
_link_class[FWY_LINK_MASK]  = "freeway"
_link_class[RAMP_LINK_MASK] = "ramp"
_link_class[SURFACE_LINK_MASK] = "surface"
_link_class[gdf_links[FT_COL].isin({70, 80})] = "transit"

# For each node, count how many distinct link classes touch it.
_stacked_class = pd.concat([
    gdf_links[["A"]].assign(cls=_link_class.values).rename(columns={"A": "N"}),
    gdf_links[["B"]].assign(cls=_link_class.values).rename(columns={"B": "N"}),
], ignore_index=True)
_class_counts = _stacked_class.groupby("N")["cls"].nunique()
gdf_nodes["LinkClassCount"] = gdf_nodes["N"].map(_class_counts).fillna(0).astype(int)

# Count distinct FT codes per node.
# Stricter than LinkClassCount: nodes where FT changes (e.g. FT=32→33) register
# LinkFTCount=2 even though both belong to the freeway class.  Used in the future
# link true-shape dissolution step to preserve model link boundaries where posted
# speed or facility sub-type changes — dissolving those nodes would corrupt TDM impedance.
_stacked_ft = pd.concat([
    gdf_links[["A", FT_COL]].rename(columns={"A": "N"}),
    gdf_links[["B", FT_COL]].rename(columns={"B": "N"}),
], ignore_index=True)
_ft_counts = _stacked_ft.groupby("N")[FT_COL].nunique()
gdf_nodes["LinkFTCount"] = gdf_nodes["N"].map(_ft_counts).fillna(0).astype(int)

# Pseudonode: shape vertex on a single homogeneous path — no physical snap target.
# Uses LinkFTCount (not LinkClassCount): UGRC/UDOT centerlines are segmented at
# posted speed limit changes, so a node where FT changes (e.g. FT=32→33) has a
# real physical endpoint in the centerline data and must be sent to snapping.
# LinkClassCount would incorrectly skip those nodes (both FT codes map to "freeway"
# class), causing missed snaps at speed-transition points along freeway mainlines.
gdf_nodes["is_pseudo"] = (gdf_nodes["NeighborCount"] == 2) & (gdf_nodes["LinkFTCount"] == 1)

print(f"Pseudonodes (NeighborCount==2, single class): {gdf_nodes['is_pseudo'].sum():,}")
print(f"  of which LinkCount==2: {(gdf_nodes['is_pseudo'] & (gdf_nodes['LinkCount'] == 2)).sum():,}")
print(f"  of which LinkCount> 2: {(gdf_nodes['is_pseudo'] & (gdf_nodes['LinkCount'] > 2)).sum():,}  ← bidirectional road pseudonodes missed by old filter")
print(f"Class-transition nodes (NeighborCount==2, multi-class): {((gdf_nodes['NeighborCount'] == 2) & (gdf_nodes['LinkClassCount'] > 1)).sum():,}  ← real junctions, must be snapped")
print(f"FT-transition nodes now sent to snapping (NeighborCount==2, same class, diff FT): {((gdf_nodes['NeighborCount'] == 2) & (gdf_nodes['LinkClassCount'] == 1) & (gdf_nodes['LinkFTCount'] > 1)).sum():,}")
```

### A3 — Node Topology Classification

Eight mutually-exclusive `node_type` labels. Transit is evaluated first and
takes priority over all road-class combinations. The remaining seven labels
derive from the full combinatorics of the (Freeway, Ramp, Surface) flags.

HOV/managed-lane nodes (FT 37-39) were filtered at load time, so they
contribute no flags; those nodes fall to "surface" and are not snapped.
Pseudonodes (LinkCount == 2) are shape vertices with no physical counterpart.

```{python}
no_pseudo = ~gdf_nodes["is_pseudo"]

is_fwy_node     = gdf_nodes["Freeway"]      & no_pseudo
is_ramp_node    = gdf_nodes["Ramp"]         & no_pseudo
is_surface_node = gdf_nodes["Surface"]      & no_pseudo
is_transit_node = gdf_nodes["FixedTransit"] & no_pseudo

# Full combinatorics of the three link classes → 7 node types.
# Transit is evaluated first — station nodes get "transit" regardless of any
# road-class links they may also touch.
# Most-specific road combinations (all three flags) are evaluated next.
gdf_nodes["node_type"] = np.select(
    [
        is_transit_node,                                  # fixed rail station
        is_fwy_node &  is_ramp_node &  is_surface_node,  # fwy+ramp+sf → at-grade interchange
        is_fwy_node &  is_ramp_node & ~is_surface_node,  # fwy+ramp    → elevated gore
        is_fwy_node & ~is_ramp_node &  is_surface_node,  # fwy+sf      → at-grade fwy crossing
        is_fwy_node & ~is_ramp_node & ~is_surface_node,  # fwy only    → pure mainline
        ~is_fwy_node & is_ramp_node &  is_surface_node,  # ramp+sf     → ramp terminal
        ~is_fwy_node & is_ramp_node & ~is_surface_node,  # ramp only   → mid-ramp
    ],
    ["transit", "gore_sf", "gore", "fwy_sf", "fwy", "ramp_sf", "ramp"],
    default="surface",
)

print("Node type distribution:")
print(gdf_nodes["node_type"].value_counts().rename("count").to_frame().to_string())
```

B1 filter to active segments → B2 classify segments → B3 extract endpoints →
B4 deduplicate → B5 assign directions → B6 assign ep_type → B7 export.

### B2 — Classify Segments

Three mutually exclusive boolean flags per segment. Primary authority is
`DOT_FCLASS` for freeway identification; `DOT_RTNAME` (UDOT LRS) supplements with
explicit route numbers and ramp/CD facility characters. `ONEWAY` in (`1`, `2`)
elevates divided Principal Arterials into the freeway class (domain CVDomain_OneWay:
0=Two-way, 1=One-way direction of arc, 2=One-way opposite direction of arc).

```
DOT_RTNAME layout (11 characters, zero-padded):
  [0:4]  Route number      e.g. "0015" = I-15, "0154" = Bangerter
  [4]    LRS direction     "P" = primary (NB/EB),  "N" = negative (SB/WB)
  [5]    Facility type     " " = mainline, "R" = ramp, "C" = collector-distributor
  [6:]   Sequence/suffix   varies
```

These flags flow into every endpoint pool and type-tier decision, making this
the **primary place to audit** when snapping results look wrong.

```{python}
# ── Controlled-access corridors in the Wasatch Front study area ───────────
# Adding / removing a route here is the primary tuning lever for freeway coverage.
# These are the 4-character UDOT route numbers (DOT_RTNAME[0:4]).
FREEWAY_ROUTES = {
    "0015",   # Interstate 15         (primary N-S spine)
    "0067",   # Legacy Parkway        (limited-access, no ramps)
    "0080",   # Interstate 80         (E-W, SLC to Parley's Canyon)
    "0084",   # Interstate 84         (Ogden corridor)
    "0089",   # US-89                 (surface in places — watch for mis-classification)
    "0092",   # Timpanogos Highway    (SR-92, American Fork Canyon)
    "0154",   # Bangerter Highway     (semi-controlled ring road)
    "0177",   # West Davis Corridor   (new build, limited segments)
    "0201",   # SR-201                (West Valley connector)
    "0215",   # Interstate 215        (belt route, SLC)
}

rtname  = active_cl["DOT_RTNAME"].fillna("")

# ── Shared exclusion flags ────────────────────────────────────────────────
# Ramp/CD character check: DOT_RTNAME LIKE '%R%' or '%C%', excluding DupRoute.
_has_R_or_C = rtname.str.contains("R", na=False) | rtname.str.contains("C", na=False)
_not_duprt  = rtname != "DupRoute"
_not_ramp_cd = ~_has_R_or_C | ~_not_duprt  # True when segment is NOT a ramp/CD

# ── Class 1: Freeway Mainline ─────────────────────────────────────────────
# Primary: DOT_FCLASS identifies interstate / other freeway geometry.
# Secondary: explicit UDOT route number is a controlled-access corridor.
# Tertiary: Principal Arterial with ONEWAY in ('1','2') represents a
#           physically separated directional roadway.
#           ONEWAY domain (CVDomain_OneWay): 0=Two-way, 1=One-way direction
#           of arc, 2=One-way opposite direction of arc.
# All three require exclusion of ramp/CD facility chars and DupRoute aliases.
_is_fclass_fwy = active_cl["DOT_FCLASS"].isin(["Interstate", "Other Freeway"])
_is_named_fwy  = rtname.str.slice(0, 4).isin(FREEWAY_ROUTES)
_is_dir_pa = (
    (active_cl["DOT_FCLASS"] == "Principal Arterial")
    & active_cl["ONEWAY"].isin(["1", "2"])
)

active_cl["is_freeway"] = (
    (_is_fclass_fwy | _is_named_fwy | _is_dir_pa)
    & ~_has_R_or_C
    & _not_duprt
)

# ── Class 2: Ramps and CDs ────────────────────────────────────────────────
# Strictly determined by DOT_RTNAME structure: any R or C facility character,
# excluding the DupRoute administrative alias.
active_cl["is_interchange"] = _has_R_or_C & _not_duprt

# ── Class 3: Surface Streets ──────────────────────────────────────────────
# Everything that is not freeway mainline or ramp/CD.
# Explicitly excludes DOT_FCLASS values that fall outside the three classes
# (already filtered in B1, but guard here for completeness).
active_cl["is_surface"] = (
    ~active_cl["is_freeway"]
    & ~active_cl["is_interchange"]
    & active_cl["DOT_FCLASS"].notna()
    & ~active_cl["DOT_FCLASS"].isin(["Interstate", "Other Freeway", "Instituitional", ""])  # note: source-data typo
)

print("Segment-level classification:")
print(f"  is_freeway     (mainline):   {active_cl['is_freeway'].sum():,}")
print(f"  is_interchange (ramp / CD):  {active_cl['is_interchange'].sum():,}")
print(f"  is_surface     (all others): {active_cl['is_surface'].sum():,}")
print(f"  unclassified   (none):       {(~active_cl['is_freeway'] & ~active_cl['is_interchange'] & ~active_cl['is_surface']).sum():,}")
print(f"  both fwy + ic  (gore segs):  {(active_cl['is_freeway'] & active_cl['is_interchange']).sum():,}")
print(f"  total active:                {len(active_cl):,}")
```

### B3 — Extract Raw Endpoints

Extract the start and end point of every active segment into a flat GeoDataFrame
(two rows per segment). Carries `FULLNAME`, `DOT_RTNAME`, and the three topology

### D1 — Freeway and Interchange Snapping

Four passes in topology-priority order: (a) pure freeway, (a2) at-grade freeway
crossings, (b) gore/gore_sf, (c) mid-ramp. All include `fwy_sf` in the D1 debug

```{python}
#
# Pass (a) — fwy nodes → fwy endpoints only.
#   Strict pool: no ramp or surface endpoints. Prevents divided-highway collapse
#   where NB and SB nodes both compete for the same non-directional endpoint.
ep_fwy_only = gdf_ep_unique[gdf_ep_unique["ep_type"] == "fwy"].copy()

#   fwy_sf nodes touch freeway links, so the freeway direction signal is reliable.
#   Surface endpoints are intentionally excluded from this pool — if no fwy-class
#   endpoint is within range the node falls through to D2 (surface pass) where
#   surface endpoints are available as a Tier-1 fallback.

    pd.concat(

            gdf_ep_unique[gdf_ep_unique["ep_type"] == "fwy_sf"],
```

### CC Attachment Nodes

Centroid connector attachment nodes are the network-side endpoints of FT_2027 = 1
(centroid connector) links — the taps where zone centroids connect to the road
network. Zone centroids themselves (N < 3600) are excluded; only the network
attachment nodes (N ≥ 3600) are retained.

These nodes are snapped to the nearest physical centerline **vertex** (not
segment endpoint) using a simple STRtree nearest-vertex lookup — no direction
filtering, no greedy assignment. The result is written as a second layer in
`nodes_snapped.gpkg` so Stage 2 can use them as additional centerline split points.

