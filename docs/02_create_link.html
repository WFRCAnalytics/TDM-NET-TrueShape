3 Part A — Resolve Snap Coordinates

nodes_snapped.gpkg stores snapped_x_round / snapped_y_round rounded to 2 decimal places (cm precision). Splitting and M-value operations require coordinates that exactly match a vertex stored in the centerline geometry. For each snapped node, this step finds the nearest unique centerline vertex within 5 cm and records its full-precision coordinate.

Show the code 
gdf_nodes_resolved = resolve_snap_coords(
    gdf_nodes_snapped,
    gdf_centerlines,
    tolerance=0.05,
)

n_resolved   = gdf_nodes_resolved["snap_resolved"].sum()
n_unresolved = n_snapped - n_resolved

4 Part B — Merge Centerlines

Dissolve all centerlines into a single geometry via unary_union, then fuse consecutive segments at degree-2 vertices via linemerge. Real intersections (3+ roads meeting) remain as line endpoints and are not merged.

Show the code 
print("Merging centerlines … (unary_union + linemerge)")
cl_merged = merge_centerlines(gdf_centerlines)

if cl_merged.geom_type == "MultiLineString":
    n_components = len(cl_merged.geoms)
elif cl_merged.geom_type == "LineString":
    n_components = 1
else:
    n_components = sum(1 for g in cl_merged.geoms if g.geom_type == "LineString")

total_length_km = sum(
    g.length for g in (cl_merged.geoms if hasattr(cl_merged, "geoms") else [cl_merged])
) / 1000
print(f"Merged: {n_components:,} components | total length: {total_length_km:,.1f} km")
Merging centerlines … (unary_union + linemerge)
Merged: 9,319 components | total length: 5,931.0 km

5 Part C — Split at Snapped Nodes

Split the merged geometry at every resolved snap point. Points that project to within 0.1 mm of a line’s existing endpoint are skipped (already a boundary). After all splits, each resulting piece is bounded exclusively by: - a snapped model node position, or - a physical road intersection preserved by linemerge.

Show the code 
print(f"Splitting {n_components:,} line components at {len(snap_points_exact):,} nodes …")
pieces = split_at_nodes(cl_merged, snap_points_exact)
print(f"Pieces after split: {len(pieces):,}")
Splitting 9,319 line components at 3,757 nodes …
Pieces after split: 9,902

6 Part D — Build Pieces Layer and Export

Extract start/end endpoint coordinates for each piece (rounded to 2 dp for matching against snapped_x_round / snapped_y_round in the next stage), then export as link_pieces.gpkg.

Show the code 
start_coords = np.array([p.coords[0] for p in pieces])
end_coords   = np.array([p.coords[-1] for p in pieces])

gdf_pieces = gpd.GeoDataFrame(
    {
        "piece_id": np.arange(len(pieces)),
        "x_start":  np.round(start_coords[:, 0], 2),
        "y_start":  np.round(start_coords[:, 1], 2),
        "x_end":    np.round(end_coords[:, 0], 2),
        "y_end":    np.round(end_coords[:, 1], 2),
        "length_m": np.array([p.length for p in pieces]),
        "geometry": pieces,
    },
    crs=CRS_UTM,
)

PATHS["out_link_pieces"].parent.mkdir(parents=True, exist_ok=True)
gdf_pieces.to_file(PATHS["out_link_pieces"], driver="GPKG")
print(f"Exported {len(gdf_pieces):,} pieces → {PATHS['out_link_pieces']}")
